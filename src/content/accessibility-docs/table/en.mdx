### WAI-ARIA Roles

| Role           | Target Element        | Description                                                                    |
| -------------- | --------------------- | ------------------------------------------------------------------------------ |
| `table`        | Container element     | Identifies the element as a table structure containing rows and cells of data. |
| `rowgroup`     | Header/Body container | Groups rows together (equivalent to `<thead>`, `<tbody>`, `<tfoot>`).          |
| `row`          | Row element           | A row of cells within the table (equivalent to `<tr>`).                        |
| `columnheader` | Header cell           | A header cell for a column (equivalent to `<th>` in header row).               |
| `rowheader`    | Header cell           | A header cell for a row (equivalent to `<th scope="row">`).                    |
| `cell`         | Data cell             | A data cell within a row (equivalent to `<td>`).                               |

### WAI-ARIA Properties

#### `aria-label`

Provides an accessible name for the table. Required for screen reader users to understand the table's purpose.

  <KeyValueListItem term="Values">String</KeyValueListItem>

</KeyValueList>

#### `aria-labelledby`

References an element that provides the accessible name for the table.

<KeyValueList>
  <KeyValueListItem term="Values">ID reference</KeyValueListItem>
  <KeyValueListItem term="Required">Yes (or aria-label)</KeyValueListItem>
</KeyValueList>

#### `aria-describedby`

References an element providing additional description for the table.

<KeyValueList>
  <KeyValueListItem term="Values">ID reference</KeyValueListItem>
  <KeyValueListItem term="Required">No</KeyValueListItem>

#### `aria-rowcount`

<KeyValueList>
  <KeyValueListItem term="Values">Number</KeyValueListItem>
  <KeyValueListItem term="Required">No</KeyValueListItem>

<KeyValueList>
  <KeyValueListItem term="Values">Number (1-based)</KeyValueListItem>

#### `aria-rowindex`

Indicates the position of a row within the full table (virtualization).

<KeyValueList>
  <KeyValueListItem term="Values">Number (1-based)</KeyValueListItem>
  <KeyValueListItem term="Required">No</KeyValueListItem>

#### `aria-colspan`

Indicates how many columns the cell spans. Only set when >1.

<KeyValueList>

Indicates how many rows the cell spans. Only set when >1.

<KeyValueList>

- The table role creates a static tabular structure for displaying data. Unlike the grid role, tables are not interactive and do not support keyboard navigation between cells.
- Keyboard support is not applicable for the table pattern. Interactive elements within cells (buttons, links) receive focus through normal tab order.

### Focus Management

| Event                | Behavior                                         |
| -------------------- | ------------------------------------------------ |
| Static table         | Not applicable - no roving tabindex needed       |
| Interactive elements | Links/buttons receive focus via normal tab order |

### Visual Design

- CSS Grid layout - Uses CSS Grid with subgrid for visual cell spanning support
- Cell borders - Gap-based borders using background color
- Sort indicators - Visual icons to indicate sort direction
- Header styling - Distinct background for header cells
