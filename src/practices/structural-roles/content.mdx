## Overview

Structural roles describe the organization of content on a page. Unlike landmark roles (which identify major sections) or widget roles (which describe interactive elements), structural roles help assistive technology understand the relationship between elements within a document.

## Common Structural Roles

| Role        | Purpose                | HTML Equivalent |
| ----------- | ---------------------- | --------------- |
| `heading`   | Section heading        | `<h1>` - `<h6>` |
| `list`      | A list of items        | `<ul>`, `<ol>`  |
| `listitem`  | An item in a list      | `<li>`          |
| `group`     | Related elements       | `<fieldset>`    |
| `separator` | Visual divider         | `<hr>`          |
| `img`       | Image content          | `<img>`         |
| `figure`    | Self-contained content | `<figure>`      |

## When to Use ARIA Structural Roles

### Use Native HTML First

Native HTML elements already have implicit roles:

<CodeFigure
  code={`<h2>Section Title</h2>
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>
<hr />`}
  lang="html"
  caption="These elements automatically receive their implicit roles: h2 → heading (level 2), ul → list, li → listitem, hr → separator."
/>

### ARIA for Custom Structures

Use ARIA when HTML semantics don't fit:

```html
<div role="heading" aria-level="2">Custom Section Title</div>

<div role="list">
  <div role="listitem">Item 1</div>
  <div role="listitem">Item 2</div>
</div>
```

## The Group Role

Use `group` to establish relationships between elements:

```html
<div role="group" aria-labelledby="address-heading">
  <h3 id="address-heading">Shipping Address</h3>
  <label>Street: <input type="text" /></label>
  <label>City: <input type="text" /></label>
</div>

<div role="group" aria-label="Text alignment">
  <button>Left</button>
  <button>Center</button>
  <button>Right</button>
</div>
```

## Heading Levels

### aria-level for Custom Headings

When using `role="heading"`, specify the level:

```html
<div role="heading" aria-level="3">This behaves like an h3</div>
```

### Avoid Skipping Levels

Maintain a logical heading hierarchy:

<CodeFigure
  code={`<h2>Main Section</h2>
<h4>Subsection</h4>`}
  lang="html"
  caption="Bad: Skipping from h2 to h4 breaks the logical heading hierarchy."
/>

<CodeFigure
  code={`<h2>Main Section</h2>
<h3>Subsection</h3>`}
  lang="html"
  caption="Good: Proper hierarchy — h3 follows h2 without skipping levels."
/>

## Separator Role

### Focusable vs Non-focusable

<CodeFigure
  code={`<hr />
<div role="separator"></div>`}
  lang="html"
  caption="Non-focusable separator — serves as a visual divider only."
/>

<CodeFigure
  code={`<div
  role="separator"
  tabindex="0"
  aria-valuenow="50"
  aria-valuemin="0"
  aria-valuemax="100"
  aria-label="Resize panels"
></div>`}
  lang="html"
  caption="Focusable separator — acts as a resizable splitter. Adding tabindex turns it into a value widget requiring range properties."
/>

## Implementation Tips

### None and Presentation Roles

See the [Hiding Semantics practice](/practices/hiding-semantics/) for when to use `role="none"` or `role="presentation"`.

### Document vs Application Mode

By default, screen readers use "document mode" for reading. The `application` role switches to "application mode" for keyboard handling—use sparingly and only when needed.

<CodeFigure
  code={`<div role="application">
  ...
</div>`}
  lang="html"
  caption="Rarely needed — custom keyboard handlers must be provided. Use with caution."
/>

## Common Pitfalls

### Removing List Semantics Accidentally

CSS can remove implicit list semantics in some browsers:

<CodeFigure
  code={`ul {
  list-style: none;
}`}
  lang="css"
  caption="This may remove list semantics in Safari."
/>

Fix with explicit role:

```html
<ul role="list" style="list-style: none;">
  <li>Still announced as a list item</li>
</ul>
```

### Overusing ARIA Structural Roles

<CodeFigure
  code={`<div role="list">
  <div role="listitem">Item</div>
</div>`}
  lang="html"
  caption="Bad: Unnecessary ARIA — native HTML elements already provide these roles."
/>

<CodeFigure
  code={`<ul>
  <li>Item</li>
</ul>`}
  lang="html"
  caption="Good: Native HTML provides the correct roles automatically."
/>

### Incorrect Heading Hierarchy

Screen reader users often navigate by headings. Ensure heading levels reflect document structure, not visual styling.
